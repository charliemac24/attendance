PURPOSE
MYO School Attendance Alerts is a multi-tenant (school-scoped), role-based school attendance + SMS notification platform. It supports QR-based kiosk check-in/out, manual attendance, per-school settings, SMS templates, and an automated absent cutoff workflow. SMS sending must use a queue job and a provider interface (default: Semaphore).

HARD REQUIREMENTS
1) Multi-tenant isolation
- All tenant data is scoped by `school_id` by default using a global scope `SchoolScope`.
- Tenant context stored in `app/Support/TenantContext.php`.
- Middleware `SetTenantContext` sets current tenant:
  - Non-super_admin: tenant = user->school
  - Super_admin: can switch tenant via `?school_id=` or session value.

2) Roles + middleware
Roles: `super_admin`, `school_admin`, `gate_staff`, `teacher`
- Implement `RequireRole` middleware.
- Access rules:
  - super_admin: global access + manage schools/users
  - school_admin: manage data within school + settings + reports + manual attendance
  - gate_staff: kiosk scanning + kiosk selector + limited today overview
  - teacher: reports limited to assigned sections (teacher_sections pivot)

3) Attendance statuses
- `pending_checkout` (checked in but not checked out)
- `late` (checked in after late time)
- `present` (checked out)
- `absent` (auto-marked after cutoff)

4) Core flows
A) Kiosk scan
- Gate staff scans QR token.
- 1st scan today => check-in (pending_checkout or late).
- 2nd scan => check-out (present).
- Extra scans blocked unless school setting `allow_multiple_scans` is true.
- Log each event in attendance_events.
- Send SMS if enabled using template tokens.

B) Manual attendance (school_admin)
- Manual check-in/out with timestamp selection.
- Optional SMS send per manual entry.

C) Absent cutoff scheduler
- Create command `mark:absent-cutoff` that runs every minute.
- For each school, when current time >= cutoff_time in that school's timezone:
  - Mark all students not checked-in today as absent
  - Log `auto_absent` event
  - Optionally queue absence SMS when enabled

IMPORT BEHAVIOR
- Parse CSV, ignore blank rows.
- Validate headers match required headers (case-insensitive).
- Validate each row:
  - Grade Level: required
  - First Name: required
  - Last Name: required
  - Student ID: required
  - Contact Number: required, normalize digits to E.164-ish for PH:
    - if starts with "0" and length 11 => convert to "63" + last 10 digits (e.g. 0917... => 63917...)
    - if starts with "63" => keep
    - if starts with "+63" => store without plus (63...)
    - if starts with "9" and length 10 => prefix "63" (917... => 63917...)
  - final stored guardian_phone should be digits only, typically 12 digits like 639171234567
- If a row fails validation, do not import it. Show it in preview with error messages.
